# 贡献指南 (Contributing Guide)

感谢你对 `cachalot` 的关注！我们非常欢迎任何形式的贡献，包括修复 Bug、改进文档或添加新特性。

## 1. 开发环境准备

- **Go 版本**：建议使用 Go 1.25+。
- **Mock 生成**：本项目使用 `mockgen`。如果你修改了接口，请运行：
  ```bash
  go generate ./internal/mocks
  ```

## 2. 仓库结构说明

- `core/`：核心抽象和通用装饰器。
- `stores/`：具体存储后端的实现。
- `examples/`：用法示例，请保持代码可编译运行。
- `docs/`：设计文档与 FAQ。

## 3. 开发流程

1. **Fork** 本仓库并创建特性分支。
2. **编写代码**：请遵循标准的 Go 代码风格，并确保添加了必要的单元测试。
3. **运行测试**：
   - 运行所有测试：`go test ./...`
   - 或者使用提供的脚本（包含子模块）：`./run_tests.sh`
4. **提交代码**：请使用清晰的提交信息。
5. **发起 Pull Request (PR)**：请在 PR 描述中详细说明你的改动动机和影响。

## 4. 提交 Issue

- 如果你发现了 Bug，请提供复现步骤。
- 如果你有新特性建议，欢迎开 Issue 讨论。

## 5. 代码风格

- 我们尽量保持 API 的简洁与一致性。
- 在添加新的 `WithXxx` 配置项时，请考虑其在 Builder API 中的易用性。
- 文档请尽量保持中英双语（README）。

---

感谢你的每一份贡献！
